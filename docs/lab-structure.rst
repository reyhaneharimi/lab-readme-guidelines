Lab README Structure (ساختار README سناریوی آموزشی)
==================================================

.. contents:: Table of Contents
   :depth: 2

1. **Title**  
   عنوان شفاف و هدف‌محور مثل:  
   `Docker - Build and Run a Container - Beginner_V1`

2. **Overview / Description**  
   خلاصه کوتاه درباره هدف Lab و ارزش یادگیری آن.

3. **Category & Level**  
   دسته‌بندی فنی (Docker, Kubernetes ...) و سطح (Beginner, Intermediate, Advanced).

4. **Estimated Time**  
   زمان تقریبی برای تکمیل سناریو (مثلاً ۳۰ دقیقه).

5. **Objectives (اهداف یادگیری)**  
   آنچه دانشجو پس از پایان Lab قادر به انجامش است.

6. **Prerequisites (پیش‌نیازها)**  
   دانش، ابزار، یا سناریوهای قبلی مورد نیاز.

7. **Scenario / Story (داستان مساله)**  
   تشریح محیط واقعی/شبه‌واقعی سناریو و اهمیت آن.

8. **Architecture Overview (نمای معماری)**  
   شرح خلاصه معماری یا دیاگرام.

9. **Step-by-Step Instructions (مراحل گام‌به‌گام)**  
   مراحل انجام کار با کد، توضیح و انتظار خروجی.

10. **Validation / Expected Output**  
    نحوه اطمینان از موفقیت انجام سناریو.

11. **Quiz & Troubleshooting**  
    سوالات تحلیلی و رفع اشکال.

12. **Common Issues**  
    مشکلات معمول و راهکارها.

13. **Hands-on Lab**  
    تمرین عملی تکمیلی (در صورت لزوم).

14. **Deliverables**  
    اقلام نهایی قابل تحویل توسط دانشجو.

15. **Summary**  
    جمع‌بندی تجربه و یادگیری.

16. **References**  
    رفرنس‌های داخلی یا خارجی.

17. **Next Steps**  
    مسیر یادگیری پس از این سناریو.

18. **Release Notes**  
    تاریخچه انتشار و تغییرات.

--  
### **Template استاندارد**

.. code-block:: markdown

   # [Category] - [Goal] - [Level] _ [Version]

   ## Overview
   ...
   (همین ساختار راهنما را به عنوان الگو با همین ترتیب بنویسید)
